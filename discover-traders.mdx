# Discover Traders

Finding the right wallets to copy is just as important as configuring your copy trading settings. Here are several ways to identify profitable traders on Polymarket.

## Merlin Strategies

​Browse curated copy trading strategies on [poly-x.trade](http://poly-x.trade). Each strategy shows a tracked wallet with its performance stats, so you can evaluate results before copying. Pick a strategy that fits your risk appetite and start copying directly from the page.

## Merlin Leaderboard

The [**PolyX Leaderboard**](https://poly-x.trade/leaderboard) ranks top-performing copy trading wallets based on real trading results. Use it as a starting point to find consistently profitable addresses.

## Selection Tips

Not every profitable wallet is worth copying. Keep these criteria in mind:

- **Consistent Gain/Loss ratio** — prefer traders with a Gain/Loss ratio above 2, indicating they win significantly more than they lose
- **Watch for suspicious curves** — a “perfect” equity curve with zero drawdown may indicate wash trading or strategies that don’t hold up when copied
- **Match your capital** — if a trader places large orders, make sure your wallet has enough balance to follow. Use **Fixed** or **Percent** mode to scale trades to your size
- **Diversify** — copy multiple traders across different categories to spread risk

<Note>
  Once you’ve found a wallet address, head to [**Copy Trading Overview**](https://docs.pollyx.trade/copy-trading/overview) to set up your config.
</Note>

## Examples

Target trades on a market at \$0.50 per share.

### Fixed (\$25) — flat amount

**Buy:** target spends \$100 →\

## Toggle Flags

Each config has three on/off flags you can toggle by tapping the corresponding button:

| **Flag** | **Default** | **What it does** |
| :-- | :-- | :-- |
| **Follow Sells** | Off | When enabled, Merlin also copies sell orders from the target wallet. When disabled, only buys are copied. |
| **Reverse Mode** | Off | When enabled, Merlin flips buy and sell. If the target buys, you sell — and if the target sells, you buy. |
| **Buy Once** | Off | When enabled, Merlin only copies the first trade per market. Subsequent trades in the same market are ignored. |

A green circle means the flag is on, red means off.

## Below Min Limit, Buy at Min

Polymarket has a minimum bet size (usually around \$1). This setting controls what happens when your calculated copy amount falls below that minimum.

- **Disabled** — if your calculated copy amount is less than \$1 (e.g., \$0.50), the bot will **skip** the trade
- **Enabled** — if your calculated amount is less than \$1, the bot will round up and force a buy of the minimum amount (approx. \$1)

<Note>
  Enable this if you are copying with a small percentage (e.g., below 5%) to avoid missing trades. It’s recommended to also set **Ignore Target Wallet Trades Under** to \$5.
</Note>

## Ignore Target Wallet Trades Under

A dust filter that helps you avoid copying insignificant or test trades. If the target buys only \$0.10 or \$1 worth of shares to “test” a market, you might not want to waste resources copying such small trades.\
\
Set a dollar threshold — the bot will ignore any trade where the target spends less than that amount.

<Tip>
  Set this to **\$5** to filter out noise while still catching meaningful trades.
</Tip>

## Advanced Filters

Tap **“Advanced Features”** on the config detail screen to reveal additional filters. These let you narrow down which trades get copied.

| **Filter** | **Format** | **What it does** |
| :-- | :-- | :-- |
| **Min Price** | 1–99 (cents), default **5c** | Only copy trades where the outcome price is at or above this value |
| **Max Price** | 1–99 (cents), default **95c** | Only copy trades where the outcome price is at or below this value |
| **Start Time** | Date/time | Only copy trades that occur after this time |
| **End Time** | Date/time | Only copy trades that occur before this time |

To clear any filter, type `clear` when prompted.

<Note>
  Price filters refer to the outcome price in cents. For example, setting Min Price to 10 and Max Price to 60 means Merlin only copies trades priced between 10c and 60c.
</Note>

## Pro Tips for Strategy

### The “Arbitrage” Setup

If you are following a high-frequency arbitrage bot, they often make small trades.

- **Recommended:** Enable **Below Min Limit, Buy at Min**
- **Warning:** This consumes more capital as it rounds up every small trade to \$1

### The “Whale” Setup

If you are following a whale who bets \$10,000 at a time.

- **Recommended:** Use **Percent** mode (e.g., 1%) and set a **Max Copy Amount** to protect your wallet from sudden massive bets

## Managing Configs

From the config detail screen, you have several management options:

- **Pause / Start** — toggle the config on or off. A paused config does not copy any trades. You must set a target wallet before you can start a config.
- **Set Target** — change the wallet address being tracked
- **Rename** — give the config a new name
- **Duplicate** — create a copy of this config with all the same settings (created in inactive state)
- **Delete** — permanently remove the config (you’ll be asked to confirm)

## Bulk Actions

From the copy trading dashboard (the list of all your configs), you can act on all configs at once:

- **Pause All** — pauses every active config
- **Start All** — starts every paused config
- **Delete All** — permanently removes all configs

Each bulk action asks for confirmation before executing.
